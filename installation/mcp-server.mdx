The MCP server brings the same YC application coaching to clients beyond Claude Code. If you use Claude.ai in the browser, work inside Cursor, or prefer ChatGPT, you can connect the YC Application Coach through the Model Context Protocol and get the same evidence-demanding, pushback-heavy coaching experience — no project directory required. Any application that supports MCP tool calls works out of the box.

## Prerequisites

<Warning>
  The MCP server requires **Node.js 18 or higher**. Run `node --version` to check. If you need to upgrade, visit [nodejs.org](https://nodejs.org).
</Warning>

## Setup

<Steps>
  <Step title="Install the package globally">
    Install `@ycroaster/mcp` from npm. A global install makes the `ycroaster-mcp` binary available system-wide so any MCP client can launch it.

    ```bash
    npm install -g @ycroaster/mcp
    ```

    Verify the installation:

    ```bash
    ycroaster-mcp --version
    ```
  </Step>

  <Step title="Add the server to your MCP client configuration">
    Open your MCP client's configuration file and add the `ycroaster` server entry. The exact file path varies by client — see the [Compatible clients](#compatible-clients) section below for client-specific locations.

    ```json
    {
      "mcpServers": {
        "ycroaster": {
          "command": "ycroaster-mcp"
        }
      }
    }
    ```

    The server runs over **stdio** (standard input/output) by default, which means it launches as a local child process. No network port is opened and no application data leaves your machine.
  </Step>

  <Step title="Restart your client and confirm the tool is available">
    Fully quit and reopen your MCP client. Look for the YC Roaster tool in the client's tool panel, tool picker, or connected servers list — the exact location depends on the client. If you do not see it, check that the binary path is correct by running `which ycroaster-mcp` in your terminal.
  </Step>

  <Step title="Start a coaching session">
    Open a new conversation and say:

    > "Help me with my YC application"

    The coach will introduce itself, ask for context about what you've built, and walk you through the application one section at a time.
  </Step>
</Steps>

## Compatible clients

<CardGroup cols={2}>
  <Card title="Claude.ai" icon="comment">
    Add the server config through the Claude.ai MCP settings panel. The YC Roaster tool appears in your tool list once connected.
  </Card>
  <Card title="ChatGPT" icon="robot">
    Add the server as a custom MCP tool action. Compatible with ChatGPT clients that support the Model Context Protocol.
  </Card>
  <Card title="Cursor" icon="code">
    Add the `ycroaster` entry to `~/.cursor/mcp.json` (global) or `.cursor/mcp.json` in your project root (project-scoped).
  </Card>
  <Card title="Any MCP-aware client" icon="plug">
    Any application that implements the Model Context Protocol tool-call interface can connect to the server using the same JSON config snippet above.
  </Card>
</CardGroup>

## Transport mode

The server communicates over **stdio** (standard input/output) as a child process of your MCP client. This is the only configuration you need for local use. It requires no port configuration and keeps all your application content on your local machine — nothing is sent over the network.

<Info>
  The MCP server provides the same coaching experience as the Claude Code skill. The same question-by-question protocol, the same pushback on vague claims, the same refusal to generate answers on your behalf, and the same final output format — a complete `yc-application-draft.md` with all your refined answers. The only difference is how the tool is invoked: through an MCP client rather than directly inside Claude Code.
</Info>
